# minquzi — 微信聊天记录业绩提取 Skill

本 Skill 用于从微信群聊截图中识别含【离】字或【消费】字的业绩记录，按规则提取字段并输出为 Excel 表格。

---

## 一、可配置项（随时修改）

### 1.1 特殊群组倍率表 `config.yaml`

某些群组的**实际业绩**需要在默认规则基础上**再乘以一个倍率**。

```yaml
special_groups:
  - name: "郑州管城51部"
    multiplier: 10
  # 添加新群组示例：
  # - name: "某某群组"
  #   multiplier: 10
```

匹配方式：**群组名称包含**该关键字即命中（模糊匹配）。

### 1.2 群组点位映射表 `config.yaml`

每个群组的点位由用户手动配置：

```yaml
group_points:
  "天津zz26南开和平门店（五日）": 0.3
  "广深杭果兮": 0.3
  "郑州管城51部": 0.25
  # 添加新群组示例：
  # "某某群组": 0.3
```

匹配方式：**群组名称精确匹配**（包含括号和特殊符号）。

如果群组未配置，点位留空，应结也留空。

## 二、Excel 字段定义（共 11 列）

| 列 | 说明 | 数据来源 |
|---|---|---|
| 日期 | 格式 `4月30日` | 调用时手动传入 |
| 群组 | 图片顶部群名称，去掉末尾 `(xxx)` 数字 | 图片识别 |
| 尾号 | 手机号后4位 | 图片识别（按规则提取） |
| 原始业绩 | 图片中的原始数字（可能带小数） | 图片识别（按规则提取） |
| 实际业绩 | 换算后的整数 | 计算 |
| 点位 | 群组对应的点位 | `config.yaml` 映射表 |
| 应结 | 实际业绩 × 点位 | 计算 |
| 总监 | 总监名称 | 手动录入（留空） |
| 奖励 | 奖励金额 | 手动录入（留空） |
| AI识别异常 | `是` / `否` | 自动判断 |
| 原始数据 | 图片中所有识别到的文字 | 图片识别（全部文字） |

## 三、识别与提取规则

### 3.1 群组提取

- 取图片**顶部第一行**文字
- 去掉末尾的 `(数字)`
- 示例：`天津南开和平门店(126)` → `天津南开和平门店`

### 3.2 样本筛选

只处理**符合以下任一条件**的消息段（以下称"样本"）：

1. 含【离】字（包括"离"、"离店"、"已离店"等）
2. 不含【离】字但含【消费】字（考虑 OCR 识别错误导致"离"被误识别）

### 3.3 尾号提取

在样本中按以下优先级提取**4位连续数字**作为尾号：

1. `尾号：XXXX` 格式（标签明确）
2. 段落开头的4位数字（如 `6514消120...` → `6514`）
3. 字符串中第一个连续4位数字

如果样本中没有4位数字 → 尾号留空。

### 3.4 业绩提取（优先级从高到低）

**规则 A：含【离】字时**

- 取【离】字**前面最近的数字**作为原始业绩
- 特殊情况：若【离】字**前无数字**但**后面**有【消费】，走规则 B
- 特殊情况：若【离】字前只有整数（非金额）需上下文判断

**规则 B：含【消费】字时**

- 取【消费】后的数字作为原始业绩
- 若【消费】后还有【余】或【剩】字，则**取【余】/【剩】后的数字**（表示最终结余才是业绩）

**规则 C：都不满足**

- 原始业绩、实际业绩、尾号字段留空
- `AI识别异常` 字段 = `是`
- 原始数据字段照常录入整段文字，用户人工处理

### 3.5 实际业绩换算

```
默认换算：
  - 原始业绩带小数点（如 400.0）→ 实际业绩 = 400.0 × 10 = 4000
  - 原始业绩不带小数点（如 120） → 实际业绩 = 120

特殊群组额外倍率：
  - 若群组名包含 special_groups 中的关键字
  - 实际业绩 = 默认换算结果 × multiplier
  - 例：郑州管城51部 "0048总120离" → 120 × 10 = 1200
  - 例：郑州管城51部 "xxx消400.0余0离" → 400.0 × 10 × 10 = 40000
```

### 3.6 应结计算

```
应结 = 实际业绩 × 点位
```

若点位未配置，应结留空。

### 3.7 AI识别异常判断

以下情况 `AI识别异常 = 是`，其他 = `否`：

- 含【离】字但无法提取到业绩数字
- 业绩数字格式明显异常
- 尾号无法识别

## 四、使用流程

### 4.1 目录结构

```
minquzi/
├── SKILL.md              # 本文档
├── config.yaml           # 配置表（倍率、点位映射）
├── parse_wechat.py       # 主脚本：解析 + 生成 Excel
├── requirements.txt      # Python 依赖
├── vendor/               # 存放微信截图
│   └── *.png
└── output/               # 生成的 Excel 输出目录
```

### 4.2 触发方式

当用户请求包含以下意图时触发本 Skill：

- "解析微信聊天记录截图"
- "从微信截图生成业绩 Excel"
- "识别 vendor 目录下的图片"
- "minquzi" / "民取字"

### 4.3 执行步骤（AI 代理遵循）

1. **读取配置**：`config.yaml` 加载特殊群组倍率和点位映射
2. **获取日期**：向用户询问日期（格式 `4月30日`）。如果用户一次提交多天，支持按图片组分别指定
3. **读取图片**：遍历 `minquzi/vendor/` 下所有图片，使用 Claude 视觉能力（`read` 工具）逐张识别
4. **逐张识别**：对每张图片
   - 提取顶部群组名称（去括号）
   - 提取全部文字作为 `原始数据`
   - 遍历每个样本（含【离】或【消费】的消息段）
   - 按 3.3/3.4 规则提取尾号、原始业绩
   - 按 3.5 规则换算实际业绩（含特殊群组倍率）
   - 按 3.6 计算应结
   - 按 3.7 判断 AI识别异常
5. **汇总数据**：将所有样本按顺序组织为表格
6. **展示预览**：以 Markdown 表格形式展示给用户校验
7. **生成 Excel**：用户确认后，调用 `parse_wechat.py` 生成 `.xlsx` 文件到 `output/` 目录，文件名格式 `业绩汇总_{日期}_{时间戳}.xlsx`

### 4.4 命令行用法

```bash
# 在 minquzi 目录下执行
python parse_wechat.py --date "4月30日" --input vendor/ --output output/

# 如果已经有识别好的 JSON 数据（由 AI 识别后整理），可直接传入
python parse_wechat.py --from-json data.json --output output/
```

## 五、关键注意点

1. **图片识别依赖 Claude 视觉能力**，不使用本地 OCR 库（本地 OCR 对微信字体和 emoji 识别率差）
2. **日期字段不从图片识别**，由用户在调用时传入
3. **总监、奖励**字段本 Skill 不填充，用户手动填写
4. **特殊群组倍率可叠加**：默认换算规则（小数×10）之后再应用特殊群组倍率
5. **一次可处理多张图片**，每张图片的每条样本独立成一行
6. **识别异常的行**必须录入表格，由用户人工核查，不能静默丢弃
