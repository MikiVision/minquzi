# minquzi

微信聊天记录截图业绩提取工具。

## 功能

识别 `vendor/` 目录下的微信群聊截图，提取含【离】字或【消费】字的业绩记录，按规则换算后生成 Excel 表格。

## 工作原理

**本 Skill 采用"AI 视觉识别 + Python 换算"两段式**：

1. **AI 负责识别**：使用 Claude 视觉能力读取图片，按 SKILL.md 规则提取群组/尾号/原始业绩/原始数据，整理为 JSON
2. **Python 负责换算**：接收 JSON，应用配置表（特殊群组倍率、点位映射）计算实际业绩、应结，并生成 Excel

不使用本地 OCR 库（微信字体和 emoji 识别率低）。

## 安装依赖

```bash
pip install -r requirements.txt
```

## 使用方式

### 方式一：对话式（推荐）

将微信截图放入 `vendor/` 目录，然后对 AI 说：

> 用 minquzi 解析 vendor 下的图片，日期是 4月30日

AI 会：
1. 读取 `config.yaml` 加载配置
2. 逐张识别图片
3. 展示 Markdown 预览表
4. 等你确认后生成 Excel

### 方式二：命令行（已有 JSON 数据时）

```bash
python parse_wechat.py --from-json data.json --date "4月30日" --output output/
```

JSON 格式示例见 `parse_wechat.py` 顶部注释。

## 配置修改（`config.yaml`）

### 添加特殊群组（实际业绩再乘倍率）

```yaml
special_groups:
  - name: "郑州管城51部"
    multiplier: 10
  - name: "新群组关键字"      # 模糊匹配
    multiplier: 10
```

### 修改群组点位（应结 = 实际业绩 × 点位）

```yaml
group_points:
  "广深杭果兮": 0.3           # 精确匹配群组全名
  "新群组": 0.25
```

## 业绩提取规则速查

| 场景 | 原始消息 | 原始业绩取法 |
|---|---|---|
| 含【离】字 | `7925前120.0退90.0余30.0离` | 取【离】前最近数字 → `30.0` |
| 【离】后有【消费】 | `1366已离店消费100.0` | 取【消费】后数字 → `100.0` |
| 【消费】后有【余/剩】 | `2615消1000.0扣查100.0余900.0离` | 取【余】后数字 → `900.0` |
| 仅【消费】无【离】 | `尾号:1744 消费：170.0` | 取【消费】后数字 → `170.0` |
| 无【离】无【消费】 | `6052海选挑离` | 留空，AI识别异常=是 |

## 实际业绩换算

```
默认：
  - 有小数点 (如 400.0)  → ×10 = 4000
  - 无小数点 (如 120)    → 原样 = 120

特殊群组（郑州管城51部）：
  - 默认结果再 ×10
  - "0048总120离" → 120 × 10 = 1200
  - "xxx余400.0离" → 400.0 × 10 × 10 = 40000
```

## 目录结构

```
minquzi/
├── SKILL.md           # Skill 规则文档（AI 读取）
├── README.md          # 本文件
├── config.yaml        # 配置表（倍率、点位）
├── parse_wechat.py    # 换算 + 生成 Excel 脚本
├── requirements.txt   # Python 依赖
├── vendor/            # 存放微信截图
└── output/            # Excel 输出目录（自动创建）
```

## Excel 字段

| 列 | 说明 |
|---|---|
| 日期 | `4月30日` 格式，由用户提供 |
| 群组 | 图片顶部群名（去掉末尾括号数字） |
| 尾号 | 手机号后4位 |
| 原始业绩 | 图片中提取的原始数字 |
| 实际业绩 | 换算后的整数 |
| 点位 | 从 `config.yaml` 读取 |
| 应结 | 实际业绩 × 点位 |
| 总监 | 手动填写 |
| 奖励 | 手动填写 |
| AI识别异常 | `是`/`否`，异常行在 Excel 中黄色高亮 |
| 原始数据 | 图片中所有识别到的文字 |
